# Contributing

Спасибо за интерес к проекту.

## Как предлагать изменения

1. Создайте ветку от `master`.
2. Делайте небольшие и логичные коммиты.
3. Проверьте, что приложение запускается локально.
4. Откройте Pull Request с кратким описанием:
- что изменено
- зачем изменено
- как проверить

## Локальная проверка перед PR

```bash
python -m py_compile app.py
node --check static/js/app.js
```

## Стиль

- Python: понятные функции и минимальная сложность.
- Frontend: аккуратный UX, адаптивность и читаемость.
- Не добавляйте секреты/токены в репозиторий.

## Scope

Для новых фич сначала предпочтительны простые решения (MVP), затем итеративное улучшение.
